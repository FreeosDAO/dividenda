◆ dryrun()

void freeosdivide::dryrun ( )

dryrun action.

This is analytics software. It counts estimated summary percentage of dividends to be paid This is counted on a basis of all active at this moment NFT's. NFT's which have already all rates paid (cap=1) or NFT's completely paid due to overcoming total accrued threshold value (cap=2) are also not counted. Cap=3 founders NFT's which are temporarily locked are counted as soon it will be paid again.
Additional information which NFT's are already paid will be given on the NFT's list displayed by the front-end. analytics table - displays cummulative percentage of active NFTs on a basis of cap category summary1 table - displays cumulative percentage of active NFTs on a basis of each user NFTs owner.

Precondition
required any account listed on whitelist.

proposalnew action.

Creates new proposal. The proposal (if accepted) becomes a base to mint new NFT.

Parameters
proposer- The account name of a person eligible to create new proposal.
eosaccount- The account name of future owner of new proposed NFT (investor or founder).
roi_target_cap- Type of Dividend Policy (roi): 1- 2- 3-
period_percentage- Pre-agreed percentage of weekly dividend which will be transfered to eosaccount.
threshold- Upper limit for dividend payment (if roi=1 n/a, if roi=2 cummulative limit, if roi=3 weekly limit).
rates_left- Number of weekly dividend payment left to pay (only for roi=1).
locked- Lock of payments (only for roi=3).
Precondition
requires proposer permission
Parameters
eosaccountfreeos account used to receive dividends and for identification
lockedlock dividends for selected new founders. Note: When unlock cannot be locked again.
